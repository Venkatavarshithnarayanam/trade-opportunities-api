### 2. Analyze Sector (Main Endpoint)

Analyze market opportunities for a specific sector.

**Endpoint**: `GET /analyze/{sector}`

**Authentication**: Required (X-API-Key header)

**Path Parameters**:
| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| sector | string | Yes | Sector name (alphabetic only, e.g., pharmaceuticals, technology) |

**Headers**:
| Header | Type | Required | Description |
|--------|------|----------|-------------|
| X-API-Key | string | Yes | API key for authentication |
| Client-ID | string | No | Client identifier for session tracking |

**Response**:
- Content-Type: `text/plain`
- Body: Markdown formatted market analysis report

**Status Codes**:
| Code | Description |
|------|-------------|
| 200 | Success - Report generated |
| 400 | Bad Request - Invalid sector input |
| 401 | Unauthorized - Invalid API key |
| 429 | Too Many Requests - Rate limit exceeded |
| 500 | Internal Server Error |

**Example Request**:
```bash
curl -X GET "http://localhost:8000/analyze/pharmaceuticals" \
  -H "X-API-Key: trade-api-key-2024" \
  -H "Client-ID: user-123"
```

**Example Response** (Markdown):

## Error Responses

### 400 Bad Request

**Cause**: Invalid sector input (contains numbers or special characters)

**Response**:
```json
{
  "detail": "Sector must contain only alphabetic characters"
}
```

**Example**:
```bash
curl -X GET "http://localhost:8000/analyze/pharma123" \
  -H "X-API-Key: trade-api-key-2024"
```

---

### 401 Unauthorized

**Cause**: Invalid or missing API key

**Response**:
```json
{
  "detail": "Invalid API key"
}
```

**Example**:
```bash
curl -X GET "http://localhost:8000/analyze/pharmaceuticals" \
  -H "X-API-Key: invalid-key"
```

---

### 429 Too Many Requests

**Cause**: Rate limit exceeded (5 requests per minute per client)

**Response**:
```json
{
  "detail": "Rate limit exceeded: Maximum 5 requests per minute"
}
```

**Example**:
```bash
# Make 6 requests with same Client-ID
for i in {1..6}; do
  curl -X GET "http://localhost:8000/analyze/pharmaceuticals" \
    -H "X-API-Key: trade-api-key-2024" \
    -H "Client-ID: user-1"
done
# 6th request will return 429
```

---

### 500 Internal Server Error

**Cause**: Unexpected server error

**Response**:
```json
{
  "detail": "Internal server error during analysis"
}
```

## Rate Limiting

**Limit**: 5 requests per minute per client

**Tracking**: Based on `Client-ID` header or IP address

**Reset**: Automatic after 60 seconds

**Example**:
```bash
# Requests 1-5: Success (200)
for i in {1..5}; do
  curl -X GET "http://localhost:8000/analyze/pharmaceuticals" \
    -H "X-API-Key: trade-api-key-2024" \
    -H "Client-ID: user-1"
done

# Request 6: Rate limited (429)
curl -X GET "http://localhost:8000/analyze/technology" \
  -H "X-API-Key: trade-api-key-2024" \
  -H "Client-ID: user-1"
# Returns: 429 Too Many Requests

# Wait 60 seconds, then requests work again
sleep 60
curl -X GET "http://localhost:8000/analyze/agriculture" \
  -H "X-API-Key: trade-api-key-2024" \
  -H "Client-ID: user-1"
# Returns: 200 OK
```

## Best Practices

### 1. Use Client-ID for Tracking

```bash
curl -X GET "http://localhost:8000/analyze/pharmaceuticals" \
  -H "X-API-Key: trade-api-key-2024" \
  -H "Client-ID: my-app-v1"
```

### 2. Handle Rate Limiting

```python
import time
import requests

def analyze_with_retry(sector, max_retries=3):
    for attempt in range(max_retries):
        try:
            response = requests.get(
                f"http://localhost:8000/analyze/{sector}",
                headers={"X-API-Key": "trade-api-key-2024"}
            )
            response.raise_for_status()
            return response.text
        except requests.exceptions.HTTPError as e:
            if e.response.status_code == 429:
                wait_time = 60
                print(f"Rate limited. Waiting {wait_time} seconds...")
                time.sleep(wait_time)
            else:
                raise
    raise Exception("Max retries exceeded")
```

### 3. Save Reports to File

```python
import requests

response = requests.get(
    "http://localhost:8000/analyze/pharmaceuticals",
    headers={"X-API-Key": "trade-api-key-2024"}
)

with open("pharma_report.md", "w") as f:

```

### 4. Batch Analysis

```python
import requests

sectors = ["pharmaceuticals", "technology", "agriculture"]

for sector in sectors:
    response = requests.get(
        f"http://localhost:8000/analyze/{sector}",
        headers={"X-API-Key": "trade-api-key-2024"}
    )
    
    with open(f"{sector}_report.md", "w") as f:
        f.write(response.text)
```

### 5. Error Handling

```python
import requests

try:
    response = requests.get(
        "http://localhost:8000/analyze/pharmaceuticals",
        headers={"X-API-Key": "trade-api-key-2024"}
    )
    response.raise_for_status()
    report = response.text
except requests.exceptions.HTTPError as e:
    if e.response.status_code == 401:
        print("Invalid API key")
    elif e.response.status_code == 429:
        print("Rate limit exceeded")
    elif e.response.status_code == 400:
        print("Invalid sector")
    else:
        print(f"Error: {e}")
except requests.exceptions.ConnectionError:
    print("Cannot connect to API server")
```

## Troubleshooting

### API Not Responding

```bash
# Check if API is running
curl http://localhost:8000/health

# If not running, start it
python main.py
```

### Invalid API Key

```bash
# Verify you're using the correct key
# Default: trade-api-key-2024

# Or set custom key
export VALID_API_KEYS="my-custom-key"
```

### Rate Limit Exceeded

```bash
# Wait 60 seconds before making another request
# Or use different Client-ID
curl -X GET "http://localhost:8000/analyze/pharmaceuticals" \
  -H "X-API-Key: trade-api-key-2024" \
  -H "Client-ID: different-user"
```

### Invalid Sector

```bash
# Sector must contain only alphabetic characters
# Valid: pharmaceuticals, technology, agriculture
# Invalid: pharma123, tech-sector, agri_tech

  -H "X-API-Key: trade-api-key-2024"
```
